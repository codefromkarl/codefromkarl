## 我在探索什么

把 AI 放进思考、学习、编码、写作和反馈循环里，而不是把它当作外部助手。

当模型可以起草、编码、总结和模仿，人的问题就从“能不能生产出来”转向“什么值得投入注意力、判断和责任”。

持续探索 AI 的边界：

1. **注意力与主体性**：当机器可以瞬间生成答案，什么仍然值得人持续投入注意力。
2. **人与 AI 协作**：如何让 AI 辅助工作可审查、可恢复、可推理，而不是变成黑箱。
3. **公开学习**：把项目、笔记和实验沉淀成可追踪的产物，展示思考如何随时间变化。
4. **实用工具**：构建能支撑更好 AI 工作流的基础设施，同时不把人的决策隐藏。

## 代表项目

### [ContextAtlas](https://github.com/codefromkarl/ContextAtlas)

一个关于“让 AI 协作更理解上下文”的实验。它关注检索、项目记忆、代码理解和可观测性，让 AI 工具基于更完整的项目图景工作，而不是只依赖一小段 prompt 猜测。

### [meta-harness](https://github.com/codefromkarl/meta-harness)

一个关于“把 AI 工作流优化变成证据，而不是感觉”的实验。它把 candidate、run、score、benchmark、proposal 和 trace 变成可以回放、比较和归档的产物。

### [TrendRadar-Rust](https://github.com/codefromkarl/TrendRadar-Rust)

一个 Rust 版趋势监控与 RSS 聚合内核，把分散的热榜、新闻源和订阅信息收敛成可抓取、可过滤、可排序、可存储、可输出的信息雷达。对我来说，它探索的是：在信息过载和 AI 加速内容生产的时代，如何用工具保护人的注意力，而不是让人被信息流牵着走。学习 Rust 的个人尝试。

## 我的原则

- 注意力是稀缺资源。AI 应该帮助人保护注意力，而不是消耗更多注意力。
- 自动化应该增强人的主体性，而不是让人变成机器输出的被动审核员。
- 工具应该让判断更容易发生，也更容易被审查。
- 好的工程很重要，因为价值观需要可靠系统，才能经受现实环境的冲击。
